# Automation Directory

This directory contains automation scripts and reports for the server-monitor project.

## Contents

### Scripts

- **parse-full-review.js**: Node.js script that parses full review data and generates a comprehensive markdown report.

### Reports

- **FULL_REVIEW_REPORT.md**: Auto-generated comprehensive project review report (generated by the Full Review workflow).

## Full Review Workflow

The Full Review workflow (`../.github/workflows/full-review.yml`) is a comprehensive manual audit workflow that:

1. **Frontend Checks**
   - Lints the Next.js frontend code
   - Runs TypeScript type checking
   - Builds the frontend

2. **Backend Checks**
   - Lints Python code with flake8
   - Runs security scans with bandit
   - Optionally runs backend tests
   - Validates Python imports

3. **Security Scans**
   - Trivy filesystem vulnerability scanning
   - Gitleaks secret scanning

4. **UI Snapshots** (Optional)
   - Captures screenshots of the application UI using Playwright

5. **Report Generation**
   - Generates a comprehensive markdown report
   - Creates/updates a GitHub issue with findings
   - Optionally creates a PR with the report

## Usage

To run the Full Review workflow:

1. Go to Actions tab in GitHub
2. Select "üîç Full Review - Manual Project Audit"
3. Click "Run workflow"
4. Configure options:
   - **create_issue**: Create/update GitHub issue with findings (default: true)
   - **create_pr**: Create PR with report (default: false)
   - **include_ui_snapshots**: Capture UI screenshots (default: true)
   - **include_security_scans**: Run security scans (default: true)
   - **run_backend_tests**: Run backend tests (default: true)

## Report Structure

The generated report includes:

- **Executive Summary**: Overall project health status
- **Quick Statistics**: Key metrics table
- **Frontend Analysis**: Linting, type checking, and build results
- **Backend Analysis**: Linting, testing, and build results
- **Security Analysis**: Vulnerability and secret scan results
- **UI Snapshots**: Screenshot capture status
- **Action Items**: Prioritized list of issues to address

## Script Usage

You can also run the parse script manually:

```bash
node automation/scripts/parse-full-review.js path/to/full-review-data.json
```

This will generate `automation/FULL_REVIEW_REPORT.md` based on the provided data file.
